# 성소판 19 — 저장 데이터 청사진 (SAVE DATA BLUEPRINT)
**작성: 렌 · 2026-07-07 · EP09 설계팩 2차 6/8 · 상태: 렌 초안(유키 판정으로 확정)**

> **성소판은 5인 파티의 전장 상황을 읽고 수습하는 사제 게임이다. 플레이어는 공격하지 않는다. 플레이어는 파티의 리베로다.**
> 이 문서는 **정책 문서**다 — 지금 localStorage를 구현하라는 뜻이 아니다. 무엇을 저장하고, 무엇을 세션에 두고, 무엇을 저장하면 위험한지의 경계를 먼저 긋는다.

- 관련: [16 보상](./16_PROGRESSION_REWARD_BLUEPRINT.md) · [11 마을 §1/§4](./11_VILLAGE_BLUEPRINT.md) · [18 범위](./18_CONTENT_SCOPE_BLUEPRINT.md)
- 태그: ✅확정 / 🔷후보 / ⏸️보류 / ⛔금지 · 코드 변경 없음

---

## 1. 현황 (✅ — 오해 방지)

- **SAVE 셸 이미 존재**: try/catch + 메모리 fallback + 단일 KEY. 기술 부채 낮음.
- **07 로드아웃 = 의도된 세션 휘발**: 세션 유지 · 새로고침 시 기본 복귀 · localStorage 미사용. **이는 버그가 아니라 07 카드에서 채택한 정책**이며, 장기 정책과 명확히 구분한다.

## 2. 3분류 (✅ 경계 잠금)

### 저장해야 할 것 (장기 🔷 — 정책 채택 후 단계 구현)
| 데이터 | 이유 |
|---|---|
| 해금 상태(보스/스킬) | 진행의 최소 단위 — 16 보상 어휘의 물리적 실체 |
| 설정(사운드 등) | 삶의 질 |
| 기록실 요약 | 최근 N판의 **다이제스트**(총 회복/사망/핵심 ins 문장) — 거울의 역사 |
| 로드아웃 프리셋 | **옵트인** — 명시적 "저장" 버튼으로만(암묵 자동 저장 ⛔) |
| **재화 잔액**(기도의 잔향·성소의 인장) | 21 §6 구조 채택(유키 2026-07-07) — **현재 구현 ⛔ · localStorage 확장 ⛔ 유지** · 실제 저장은 **Save Policy Revision 카드 이후** 가능 |

### 세션만 유지 (✅ 현행 정책)
현재 로드아웃(성당 저장 카드 전까지) · 전투 진행 상태 · 화면 위치. — Chapel Save/Return 01은 "세션 내 저장·복귀"까지만 확장하며 이 분류를 벗어나지 않는다.

### 저장하면 위험한 것 (⛔)
| 데이터 | 위험 |
|---|---|
| 전투 중 상태 스냅샷(이어하기) | 코어 결정론·리포트 무결성 훼손, 재현 불가 버그의 온상 |
| 리포트 원문 전량 | 용량·파싱 부채 — 다이제스트만 저장 |
| 코어 내부 구조 직렬화 | 코어 버전에 데이터가 종속 — **저장은 어댑터 데이터만** |
| 개인 정보·식별자 일체 | 게임 데이터만 저장한다(데이터 최소주의) |

## 3. 기술 원칙 (✅ 셸에 이미 절반 구현 · 나머지 🔷)

1. try/catch + **메모리 fallback** — 저장 실패가 플레이를 막으면 ⛔ (조용히 세션 모드로).
2. 단일 KEY 네임스페이스 + **스키마 버전 필드** 🔷 — 로드 실패·버전 불일치 시 **무시하고 새 시작**(게임 진행 무손상 원칙).
3. 쓰기 시점 = 화면 전환 등 **명시적 이벤트만**. 전투 프레임 루프 중 쓰기 ⛔.
4. **리셋 제공** 🔷 — 플레이어가 언제든 지울 수 있다.
5. 외부 전송·클라우드 ⛔ — 단일 HTML·오프라인 완결 원칙.

## 4. 07 정책 vs 장기 정책 (✅ 구분표)

| 단계 | 로드아웃 | 해금/기록 | 상태 |
|---|---|---|---|
| 현재(07) | 세션 · 새로고침 기본 복귀 | 없음 | ✅ 의도된 정책 |
| Chapel Save/Return 01 | **세션 내** 저장·마을 복귀 | 없음 | 🔷 다음 카드 |
| 장기(본 문서 채택 후) | 프리셋 옵트인 저장 | 해금+설정+기록 다이제스트 | ⏸️→🔷 (유키 채택 필요) |

## 5. 금지 (⛔)

정책 채택 전 localStorage 확장 · 전투 중 저장 · 암묵 자동 저장(프리셋) · 코어 직렬화 · 개인 정보 저장 · 외부 전송.
