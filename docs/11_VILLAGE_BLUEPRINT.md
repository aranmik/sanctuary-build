# 성소판 11 — 마을 청사진 (VILLAGE BLUEPRINT)
**작성: 렌 · 2026-07-07 · EP08 설계팩 4/6 · 상태: 렌 초안(유키 판정으로 확정)**

> **성소판은 5인 파티의 전장 상황을 읽고 수습하는 사제 게임이다. 플레이어는 공격하지 않는다. 플레이어는 파티의 리베로다.**
> 성소판의 마을은 **"다음 전투의 답을 준비하는 곳"**이다.
> 원형의 교훈(조사노트00): 레이드의 절반은 전투 전에 있었다 — 배분표, 버프, 소모품, 담당 지정. 성소판은 그 "준비가 곧 플레이"인 감각을 1인 게임으로 번역한다. 단, **노동의 복제는 ⛔** — 준비는 선택이어야지 숙제가 되면 안 된다.

- 관련: [08 플로우](./08_GAME_FLOW_BLUEPRINT.md) · [09 스킬](./09_PRIEST_SKILL_BLUEPRINT.md) · [10 동료](./10_COMPANION_BLUEPRINT.md) · [12 로드맵](./12_SANCTUARY_SYSTEM_ROADMAP.md)
- 태그: ✅확정 / 🔷후보 / ⏸️보류 / ⛔금지 (08 공통 범례)

## 1. 성당/성소 — 답을 저장하는 곳

| 기능 | 상태 | 메모 |
|---|---|---|
| 사제 스킬 장착(6슬롯) | ✅ (07 미니) | 8스킬 중 6개, 풀 순서 렌더, 정화 경고(잠금 아님) |
| 스킬 설명 | ✅ 부분 | `SK_DESC` 존재 — 성당 내 노출 강화는 🔷 |
| **저장하고 마을로 복귀** | 🔷 | **장기 정답.** 현재의 "고르고 바로 전투"는 임시(07 명시) |
| 전투 시작 버튼 | ✅ 임시 | 길드 보드 동선 완성(08A/08B) 후 제거 예정 |
| 스킬 강화 | ⏸️ | 강화 셸(Skill Upgrade Shell)부터 — 수치 인플레 경계 |
| 패시브 장착 | ⏸️ | 03 풀의 패시브 규칙 확정 선행 (09 §4-6~8) |
| 신성력 스킬 안내 | 🔷 | "6칸 안의 선택지" 원칙을 성당 UI 문구로 |

- 로드아웃 저장 방식(세션 초월 프리셋·localStorage 정책)은 ⏸️ 별도 결정 — SAVE 셸(try/catch+메모리 fallback)은 이미 존재하므로 기술 부채는 낮다.

## 2. 길드 보드 / 토벌 게시판 — 문제를 고르는 곳

| 기능 | 상태 | 메모 |
|---|---|---|
| 몬스터 선택 | 🔷 (씨앗 ✅) | `guild`+`BOSSES` 구조 존재, 현재 모르가스 1종 |
| 위험 태그 표시 | 🔷 (씨앗 ✅) | `BOSSES.tags` 이미 존재 — 08 문서 §4의 태그 어휘 v1로 승격 |

| 추천 준비 표시 | 🔷 | "정화 권장 / 광역 치유 권장 / 지속 피해 많음 / 차단 중요 / 사제 주시 위험" 등 — 태그의 답 안내 |
| 현재 로드아웃과의 위험 경고 | 🔷 | 태그 요구 vs 장착 대조. **경고만, 잠금 아님**(07 정화 경고 철학 계승) ⛔ 잠금 |

## 4. 훈련장 / 기록실 — 거울을 보관하는 곳

| 기능 | 상태 | 메모 |
|---|---|---|
| 지난 전투 리포트 보기 | 🔷 | 리포트 아카이브 — 저장 정책(⏸️)과 연동 |
| 스킬별 기여도 확인 | 🔷 | 06 ledger 구조가 이미 스킬별 분해의 씨앗 |
| 실패 원인 힌트 | 🔷 | "이번 판 나는 무엇을 놓쳤나?" — `ins` 인사이트의 아카이브화 |

- 기록실의 존재 이유는 성장 서사다: 같은 보스를 다시 잡을 때 지난 거울을 꺼내 볼 수 있으면, 리포트가 1회용에서 **여정**이 된다.

## 5. 장비 / 성물 / 유물 (⏸️ — 조기 구현 금지)

- **원칙**: 수치 강화(+10% 치유)보다 **선택 방식을 바꾸는 유물** 우선. 숫자는 손맛을 인플레로 잠식하지만, 선택 변경은 로드아웃 고민을 확장한다.
- 예시(전부 ⏸️ 아이디어 단계):
  - 소생 지속 +1틱 — 예약의 길이를 늘리는 선택
  - 기도씨앗 전이 우선순위 변경 — 예측의 문법을 바꾸는 선택
  - 정화 후 짧은 보호막 부여 — 심장 연계에 리듬 추가
- ⛔ **성소판 기본 손맛을 깨는 유물 금지** — 유물 없이도 완전한 게임이어야 하고, 유물은 "다르게"를 주지 "더 세게"를 주지 않는다.
- 원형 참고: 조사노트02의 축복/문장 경제(준비 의식), 04의 유물적 특성 감각 — 단 소모품 노동(양초·문장 수급)의 복제는 ⛔.

## 6. 마을 전체 플로우 (08 문서 §3의 마을 시점 뷰)

```
[성당] 고르고 저장 → [마을] → [길드 보드] 몬스터·태그·보상 확인
   → [출정 확인] 답(로드아웃)과 문제(태그) 대조 → [전투]
   → [리포트] → [보상/해금(⏸️)] → [마을] (기록실에 거울 적재 🔷)
```
