# 38. 성당 스킬 장착 UI (CHAPEL LOADOUT UI)

**작성: 렌 · 2026-07-09 · EP23 · 상태: 구현 카드(PASS 후보 — 나라님 폰 실기 대기) · 패치 p19 · 어댑터/UI 전용(CORE 무접촉)**
**상호 링크**: [37 마을 UX 계약](./37_VILLAGE_UX_CONTRACT.md)(§B·C·D·E 근거) · [35 출격 거울](./35_SORTIE_WARNING_COPY_POLISH.md) · [34 결과 거울](./34_BOSS_SPECIFIC_REPORT_HINT.md) · [07 성당 로드아웃 원형](./07_CHAPEL_LOADOUT_MINI.md)

> **핵심 문장 3줄**
> 1. 성당을 "스킬 목록 화면"에서 **"내가 이번 전투를 어떤 6개 수습 도구로 준비하는지 항상 읽을 수 있는 공간"**으로 바꾼다.
> 2. 장착 6칸을 **상단 고정(sticky)** — 스크롤해도 안 사라진다. 스킬은 **4카테고리 탭**으로 나눠 20스킬 확장에 대비.
> 3. 성당은 준비 공간이다 — **성당 내 전투 시작 버튼 제거**. 출격은 토벌 게시판/보스 선택 흐름에만.

---

## 1. 목적

현행 문제(37 §B): 성당에서 스킬 목록을 스크롤하면 장착 6칸이 화면 밖으로 사라져, "무엇을 꼈고 무엇이 빠졌는지" 보면서 후보를 비교할 수 없다. 9스킬에선 버티지만 20스킬 전후에선 판단 불가. 본 카드는 [37]의 성당 설계(§C 레이아웃 · §D 카테고리 · §E 메시지)를 첫 구현한다.

## 2. docs/37과의 연결

- §C 목표 레이아웃 → 상단 고정 슬롯 + 구성 요약 + 카테고리 탭 + 카테고리별 목록 + 저장 버튼(전투 시작 버튼 제거).
- §D 카테고리 4종(전장 문제 기준) → 데이터 테이블 `SKILL_CAT`로 구현.
- §E 메시지 3단 분리 → 이번 카드는 **성당 단계(내 구성의 일반 성향)**만 구현. 출격(35)·결과(34)는 이미 존재.

## 3. 장착 슬롯 상단 고정 방식

- `#chapel-skills` 안을 두 블록으로 재구성: **`#chapel-fixed`**(sticky) + **`#chapel-list`**(스크롤 대상).
- `#chapel-fixed{position:sticky;top:0;z-index:2;background:var(--parch);box-shadow:...}` — 스크롤 컨테이너(`.twScroll`) 상단에 붙어, 스킬 목록이 그 아래로 흐른다.
- `#chapel-fixed` 안에 순서대로: 장착 카운트(`장착 N / 6`) → 슬롯 6칸(`.ldSlots`) → 구성 요약(`.ldSum`) → 카테고리 탭(`.ldCat`).
- **슬롯 상태 3종**: `.ldSlot.empty`(비어 있음 · 점선) / `.ldSlot.filled`(장착됨 · 아이콘) / `.ldSlot.filled.sel`(선택 중 = 현재 탭 카테고리에 속한 장착 슬롯 · 금색 강조). 탭↔슬롯이 시각으로 연결된다.
- 기존 클래스(`.ldSlots`/`.ldSlot`/`.ldHead`/`.ldCount`) 재사용 · 신규 CSS 최소(sticky/탭/요약/sel = 9줄).
- 검증: 모바일 375px에서 가로 overflow 0 · 스크롤 200px 후에도 슬롯이 시야에 유지(`fixed.top == scroll.top`).

## 4. 카테고리 탭 구조

- `SKILL_CAT`(data-driven 테이블) + `CHAPEL_CATS`(탭 4종) + `CHAPEL_CAT`(세션 활성 탭 · 저장 안 함).
- 탭 클릭 → `chapelCatSet(c)` → 활성 카테고리 스킬만 `#chapel-list`에 표시 + 슬롯 sel 갱신.
- 스킬 카드는 기존 구조(`.twSkill.ldSel` · 아이콘/이름/비용/설명/체크) 그대로 재사용 — 클릭 시 `loadoutToggle` (장착/해제) 무변.

## 5. 현재 9스킬 분류표 (전장 문제 기준 · 빈 카테고리 없음)

| 카테고리 | 스킬 | 문제 |
|---|---|---|
| **치유** | 빠른 치유(heal) · 강력 치유(big) · 치유의 기도(pray) | 직접 HP 복구 |
| **보호** | 보호막(shield) · 긴급 구원(save) | 피해가 들어오기 전/치명 피해를 받침 |
| **정화·유틸** | 정화(dispel) | 상태 사고 · 행동 기회 · 전장 꼬임을 풂 |
| **지속·자원** | 소생(renew) · 기도씨앗(seed) · 성심 집중(focus) | 장기전 · 마나 · 유지력 · 버티기 |

- 긴급 구원(save)은 [37 §D]에서 "치유/보호 경계"로 남았던 항목 — 본 카드에서 **보호**로 확정(치명 피해를 받쳐 죽음을 막는 성격). 4탭 균형: 치유 3 / 보호 2 / 정화·유틸 1 / 지속·자원 3.
- 20스킬 확장 시 각 스킬에 `SKILL_CAT[key]` 한 줄만 추가하면 자동 편입(data-driven).

## 6. loadout 구성 요약 정책 (성당 단계 = 일반 성향)

- `chapelSummary()` — 현재 LOADOUT의 **일반 성향**을 최대 3줄. **보스별 정답지가 아니다**(보스 어휘 0). ← 기존의 보스 어휘("정화 없음 — 낙인/차단 연계가 어려워집니다") 제거하고 성향 문장으로 교체.
- 규칙(약점 먼저 → 강점):
  - 정화 없음 → "정화 없음 — 상태 사고 대응이 약합니다."
  - 보호막 없음 → "보호 부족 — 큰 단일 피해를 미리 받치기 어렵습니다."
  - 소생·기도씨앗 둘 다 없음 → "지속 회복 부족 — 장기전에서 마나 부담이 커질 수 있습니다."
  - 강력 치유 있음 + 지속 없음 → "큰 치유 과다 — 순간 복구는 강하지만 장기전 부담이 커질 수 있습니다."
  - 치유의 기도 있음 → "광역 회복 있음 — 파티 전체 흔들림을 수습할 수 있습니다."
  - (아무 신호 없으면) → "구성이 균형 잡혀 있습니다."
- **3단 분리 완성**: 성당(일반 성향 · 본 카드) / 출격(보스 위험 · [35]) / 결과(실제 흔들림 · [34]).

## 7. 성당 내 전투 시작 버튼 처리

- **제거**: `▶ 이 구성으로 전투 시작`(chapel-start) 버튼 + `enterBattle(1)` 임시 출격 경로 완전 삭제.
- **유지**: `🏛️ 저장하고 마을로`(chapel-save → chapelSaveReturn).
- **출격 흐름 무손상**: 정식 출격(`sortieGo → enterBattle(SELECTED_BOSS)`)과 길드 보스 선택은 그대로 — 브라우저 실증(길드 3보스 카드 · 출정 버튼 · 전투 진입 정상).
- 안내 문구도 정리: 성당 하단 note에서 "이 구성으로 출정합니다"(성당 출격 오인) 제거 → "출정은 토벌 게시판에서".

## 10-B. 나라님 실기 피드백 반영 (v2 · p19b · 같은 카드 내 수정)

큰 방향(슬롯 상단 고정·카테고리 탭)은 실기에서 "매우 좋음" 확인. 커밋 전 4항목 다듬음:

1. **선택 카드 가독성** — `.twSkill.ldSel.on` 배경을 **다크 그린(#16351f→#0f2417) → 라이트 그린 틴트(#e6f5e8→#d6eed9)**로 교체. 선택 표현은 배경 전체 대신 **좌측 강조선(inset 3px green) + 초록 이름색(#1f5b2c) + 체크(✓)**로 분산. 텍스트(이름/설명)가 다시 선명하게 읽힘("선택됨"은 보이되 읽기 희생 없음).
2. **마나/신성력 자원 구분** — 비용 표기에 자원별 클래스+아이콘: **마나 = 💧 파란색(#2f6fd0)** / **신성력 = ✦ 금색(#c98f1d)**. 한눈에 다른 자원으로 보임. (기존 비용 표기 UI의 시각 구분만 — 새 자원 시스템 0.)
3. **저장 버튼 하단 고정** — `🏛️ 저장하고 마을로`를 스크롤 컨테이너(`.twScroll`) **밖의 `#chapel-foot`(flex:none·상단 border)**로 이동. **3존 확정: 상단 고정(#chapel-fixed 슬롯/탭) · 중단 스크롤(#chapel-list) · 하단 고정(#chapel-foot 저장)**. 어떤 카테고리를 선택하든 저장 버튼은 항상 같은 위치. (리스너는 bindings에서 1회 부착 · chapelSaveReturn 총 2회 유지.)
4. **구성 요약 높이 고정** — `.ldSum{min-height:50px}` — 3줄 전용 공간 상시 확보. 요약이 1줄이든 3줄이든 높이 50px 고정 → **스킬 선택 시 레이아웃 점프 제거**(브라우저 실측: 1줄 케이스 50px == 3줄 케이스 50px). 성당 메시지는 여전히 "일반 성향"만(보스 어휘 0).
