## A. 전투 화면 시각 목표 (✅ 잠금)

성소판 전투 화면의 목표:

- **전투는 텍스트만 읽는 화면이 아니다.** 텍스트는 확정 정보를 주지만, 전장의 공간감·호흡·위험의 방향은 시각이 준다.
- **보스와 동료는 서로 다른 객체로 보여야 한다.** 지금 화면에서 모르가스와 파쇄자와 심연이 "다른 존재"로 읽히지 않으면, 세 질문(판단/화력/인내)이 손보다 먼저 눈에서 구분되지 않는다.
- **행동선은 전투의 호흡이자 문법이자 힌트다.** 보스가 다음에 무엇을 하는지, 그것이 전장 어디를 흔드는지, 파티의 연결이 살아있는지 — 선이 말한다.
- **텍스트 예고와 CSS 행동선은 서로 보완한다.** 어느 쪽도 다른 쪽을 대체하지 않는다(§B).
- **사제는 화면 위의 피규어/행동선/상태를 보고 전장을 읽는다.** 읽기(예보·순서) → 고르기(대상·스킬·타이밍) → 수습([33 §6] 리베로 원칙)의 "읽기" 단계를 시각이 책임진다.

> **`보스 다음 행동` 텍스트는 테트리스의 다음 블록처럼 유지되어야 한다.**
> **CSS 행동선은 그 정보를 전장 위에서 읽게 만드는 보조 힌트다.**

## B. 텍스트 예고와 행동선의 관계 (✅ 잠금)

현재 성소판에는 "보스 다음 행동: 강타" 같은 **텍스트 예고**가 있다. 이 텍스트 예고는 **반드시 유지한다** — 제거는 ⛔(§K).

**원칙 5줄**:

1. **텍스트 예고는 확정 정보다.** 다음에 무엇이 오는지는 문자로 못 박는다(결정론·난수 0과 정합).
2. **행동선은 그 정보가 전장 어디에 영향을 주는지 보여준다.** "강타"라는 단어를, 보스에서 전사로 향하는 선으로 공간화한다.
3. **행동선은 텍스트를 대체하지 않는다.** 선만 남기고 텍스트를 빼는 순간, 확정 정보가 해석 정보로 격하된다 ⛔.
4. **행동선은 정답 버튼을 가리키지 않는다.** 사제의 판단(어떤 스킬로 수습할지)은 사제의 몫이다 — 선이 대신 고르면 리베로가 죽는다.
5. **행동선은 "어디가 흔들릴 수 있는지" 보여준다.** 문제의 위치를 보여주되, 답을 보여주지 않는다.

**좋은 예**:
- 보스 → 전사에게 굵은 압박선 (강타가 어디로 가는지)
- 낙인 대상과 도적 차단선의 연결/끊김 (판단 문제가 어디서 꼬였는지)
- 파티 전체에 얇은 갈증/누수선 (자원이 어디서 새는지)

**나쁜 예** ⛔:
- "지금 보호막을 누르세요"
- "정답은 정화입니다"
- 버튼을 직접 강조해서 정답을 알려줌
- 예고 없이 즉사성 정보만 숨김 ([33 §5] 억까 판별 1기준 위반)

## C. 친절도 단계와 튜토리얼 3보스 (✅ 나라 판단 반영 · 잠금)

**나라 판단**: 현재 3보스(모르가스 · 강철의 파쇄자 · 갈증의 심연)는 나라 기준에서 **튜토리얼에 가깝다**.

**튜토리얼 3보스 원칙**:
- 튜토리얼 3보스는 **친절해도 된다**.
- 출격 경고([35]) · 다음 행동 텍스트 · 결과 거울([34])이 명확해도 된다.
- 이들은 성소판의 **기본 문법을 가르치는 보스**다 — 정화/차단(판단), 보호막/버티기(화력), 절약/유지(인내).

**향후 일반/심화 보스 원칙**:
- 정보를 전부 알려주지 않아도 된다.
- 하지만 **배운 문법으로 추론할 수 있어야 한다** — 튜토리얼에서 가르친 어휘(압박선/연결선/누수선)가 새 보스에서도 같은 의미로 쓰이면, 텍스트가 줄어도 전장이 읽힌다.
- **정답은 숨겨도 되지만 문제의 종류는 보여줘야 한다.** "이건 연결이 꼬이는 보스다"까지는 보이고, "그래서 언제 무엇을 정화할지"는 사제가 푼다.
- **불친절은 허용하지만 억까는 금지한다** — [33 §5] 3기준(예고 또는 개연 / 반응 창 / 수습 수단)은 친절도와 무관하게 모든 보스에 적용된다.

> **튜토리얼 보스는 답을 가르친다.**
> **이후 보스는 질문을 던진다.**
> **변주 보스는 배운 답이 흔들리는 순간을 만든다.**

## D. 보스별 시각 정체성 (✅ 잠금 — 3보스 = 3실루엣 = 3어휘)

세 보스는 [세이브 §4-D]의 세 질문(판단/화력/인내)이다. 화면에서도 세 개의 **다른 객체**로 읽혀야 한다.

### D-1. 핏빛 예언자 모르가스 — 꼬는 자

- **역할**: 판단 · 정화 · 차단 · 상황 수습. **핵심**: 낙인 정화 → 도적 차단 성공.
- **시각 방향**: 가늘고 비틀린 실루엣 · 저주/낙인/문양 · **끊어진 선과 이어지는 선**. 전장을 직접 때리는 보스가 아니라, **전장의 연결을 꼬는 보스**처럼 보여야 한다.
- **행동선 방향**:
  - 보스 → 낙인 대상에게 **어두운 낙인선**.
  - 낙인 상태에서 도적의 차단선이 **흐려지거나 끊김** (문제가 시각으로 보임).
  - 정화 후 도적 차단선이 **다시 살아남** (수습이 시각으로 보임).
  - 차단 성공 시 보스 행동선이 **끊기는** 피드백 (기존 로그 문법과 짝).
- **키워드**: `연결 / 차단 / 정화 / 복구 / 꼬임`

### D-2. 강철의 파쇄자 — 부수는 자

- **역할**: 화력 · 탱커 버티기 · 보호막 · 원초적 레이드 체크. **핵심**: 전사는 버티고, 사제는 살리고, 딜러는 고통을 빨리 끝낸다.
- **시각 방향**: **큰 사각형** · 무거운 금속 실루엣 · 망치/갑옷/각진 형태. 보기만 해도 **"정면 압박이 온다"**는 느낌. 모르가스의 가늘고 비틀림과 정반대 축.
- **행동선 방향**:
  - 보스 → 전사에게 **굵은 강타선**.
  - 강타 예고가 **점점 응축됨** (예고 텍스트의 시간 정보를 선의 밀도로 번역).
  - 보호막이 있으면 선이 **방패층에 걸림** (보호선 §G-4와 상호작용).
  - 보호막이 없으면 전사에게 **그대로 꽂힘**.
- **키워드**: `압박 / 충돌 / 강타 / 전선 / 버티기`

### D-3. 갈증의 심연 — 마시는 자

- **역할**: 인내 · 마나 압박 · 절약 운영 · 생존 버티기. **핵심**: **무엇을 안 쓸 것인가?**
- **시각 방향**: 액체/심연/안개/**흡수체** · 낮고 퍼지는 형태. 직접 때리는 느낌보다 **계속 빨아들이는 느낌** · 생기가 빠져나가는 분위기. 파쇄자의 응축과 정반대 — 퍼지고 스민다.
- **행동선 방향**:
  - 파티 전체에 **얇은 갈증선** (한 명이 아니라 모두에게).
  - 사제 주변 **자원이 말라가는** 표현.
  - 큰 치유는 순간 밝지만 **마나가 크게 빠지는 느낌** (비용이 시각으로 보임).
  - 소생/기도씨앗/성심 집중은 **작지만 오래 이어지는 유지선** (절약 운영의 시각 보상).
- **키워드**: `소모 / 누수 / 마나 / 유지 / 버티기`

**세 실루엣 대비 요약**: 모르가스 = 가늘고 비틀림(선을 꼬는) · 파쇄자 = 크고 각짐(선을 꽂는) · 심연 = 낮고 퍼짐(선을 빠는). 세 보스가 같은 화면 문법 안에서 **다른 동사**로 읽힌다.

## E. 동료/직업 미니 피규어 문법 (✅ 잠금)

동료는 단순 아이콘이 아니라, **전장 위의 미니 피규어**처럼 읽혀야 한다.

**원칙**:
- **모바일에서 읽히는 실루엣이 먼저다.** 390px에서 실루엣이 구분되지 않으면 실패(§H).
- **얼굴 디테일보다 역할 실루엣이 중요하다.**
- 직업/역할은 **몸통 · 손 오브젝트 · 받침 · 역할 효과**로 구분한다.
- 피규어는 **행동선과 상태 표시를 방해하지 않아야 한다** — 피규어는 무대의 배우이지 무대 자체가 아니다.
- **귀여움보다 읽기 우선이다.**

**조립형 구조 후보** (부품 계층 · §F와 짝):

| 부품 | 역할 |
|---|---|
| 바닥/받침대 | 객체의 존재감 · 위치 앵커 |
| 몸통 | 역할의 부피(탱커 넓게 · 딜러 가늘게) |
| 머리/헬멧/후드 | 직업 1차 식별 |
| 손 오브젝트 | 직업 2차 식별(방패/칼/토템/활) |
| 역할 장식 | 오라·문양 등 최소 장식 |
| 상태 오버레이 | 낙인/중독/버프 — 피규어 위에 얹힘 |
| 행동선 앵커 | 선이 출발/도착하는 지점 |

**직업별 실루엣 예시**:
- **전사**: 넓은 어깨 · 방패/무기 · 낮은 중심 · 단단한 받침 — "강타선이 꽂히는 곳".
- **도적**: 가는 실루엣 · 짧은 칼 · 빠른 자세 · **차단선 앵커** — "연결선이 출발하는 곳".
- **주술사**: 토템/구슬/지팡이 · 자원 유지 오라.
- **원거리 딜러**: 활/투사체 방향성.
- **사제**: 전장 중앙을 읽는 지원자 — **직접 공격자가 아님**(성소판 정의와 정합).

**주의** ⛔:
- 이번 카드에서 동료 7종을 구현하지 않는다.
- 7명 확정 동료 방향은 [37 §G]를 따른다.
- 다중 영웅 모집은 장기 후보로만 남긴다.

## F. 조립형 CSS / 브릭 토이 원칙 (✅ 잠금)

나라가 제안한 "레고 블록처럼 조립형" 방향을 성소판식으로 잠근다. 단, 특정 상표/브랜드의 구현이 아니라 **브릭 토이 / 조립형 미니 피규어 문법**으로 일반화한다.

**원칙**:
- 피규어는 **CSS 부품 조합**으로 구성될 수 있다 (§E 부품 계층).
- 보스/동료는 완전히 새로 그리기보다 **부품 조합으로 확장 가능**해야 한다.
- **같은 몸체 + 다른 장식 = 변주 보스** — [33 §6]의 "기준 택틱 + 찰나의 사고 하나" 문법과 시각이 정확히 짝을 이룬다. 변주 보스가 기준 보스의 몸을 공유하면, 플레이어는 "아는 보스인데 뭔가 다르다"를 눈으로 먼저 감지한다.
- **같은 직업 기반 + 다른 손 오브젝트/오라 = 장기 확장** (다중 영웅 후보가 살아나도 문법 재사용).
- 단, 모바일 가독성을 위해 **한 캐릭터의 핵심 읽기 포인트는 2~3개로 제한**한다.

**금지 방향** ⛔:
- 디테일 과다.
- 작은 얼굴 장식 의존.
- 행동선보다 피규어가 더 시끄러워짐.
- CSS 대공사 전제.
- 한 번에 모든 직업/보스 적용.

## G. 행동선 종류 (✅ 잠금 — 5종 분류)

성소판의 행동선 어휘. 각 선은 **목적 / 쓰임 / 피해야 할 점**을 갖는다.

### G-1. 압박선
- **목적**: 큰 단일 피해의 방향과 무게를 보여준다.
- **쓰임**: 파쇄자 강타 — 보스 → 전사. **굵고 무겁고 직선적.** 예고 응축(시간이 다가올수록 밀도↑).
- **피해야 할 점**: 매 틱마다 깜빡여 소음이 되는 것. 압박선은 "다음에 온다"의 선이지 "지금 맞았다"의 FX가 아니다.

### G-2. 연결선
- **목적**: 객체 사이의 관계(차단·낙인)가 살아있는지/끊겼는지 보여준다.
- **쓰임**: 모르가스 낙인/차단 — **이어짐/끊김/복구**가 핵심. 도적 차단선이 흐려지면 "판단 문제 발생", 다시 이어지면 "수습 완료".
- **피해야 할 점**: 연결선이 너무 많아 거미줄이 되는 것. 의미 있는 연결(현재 문제의 축)만 그린다.

### G-3. 누수선
- **목적**: 지속적·전역적 소모(마나 압박·약광역)를 보여준다.
- **쓰임**: 심연 갈증 — 파티 전체에 **얇고 지속적이며 서서히 말라가는** 선.
- **피해야 할 점**: 얇은 선을 굵게 키워 압박선과 혼동시키는 것. 누수는 무게가 아니라 **지속**으로 읽혀야 한다.

### G-4. 보호선
- **목적**: 준비가 피해를 받아내는 순간을 보여준다.
- **쓰임**: 보호막이 강타를 받치는 순간 — 압박선이 **방패층에 걸리거나 완충**됨. "준비했다면 수습 가능"([33 §7])의 시각 보상.
- **피해야 할 점**: 보호선이 "보호막을 켜라"는 사전 지시가 되는 것 ⛔ — 보호선은 **결과**(받아냈다)이지 **지시**(눌러라)가 아니다.

### G-5. 수습선
- **목적**: 사제의 개입(치유/정화/유지)이 전장을 다시 세우는 것을 보여준다.
- **쓰임**: 정화 → 끊긴 차단선 복구 · 소생/씨앗 → 유지선 · 치유 → 생기 복귀. **정답 버튼 강조가 아니라 수습 결과를 보여줌.**
- **피해야 할 점**: 수습선이 화려해져 "스킬 이펙트 쇼"가 되는 것. 수습선의 주인공은 스킬이 아니라 **다시 선 전장**이다.

**어휘 재사용 원칙**: 5종은 보스가 늘어도 재사용된다 — 새 보스는 새 선 종류가 아니라 **기존 5종의 새 조합**이 기본이다(새 종류 추가는 본 계약 개정 필요).

## H. 정보 과잉 방지 원칙 (✅ 잠금)

전투 화면은 **예뻐지는 것보다 읽히는 것이 먼저다.**

- **한 순간에 강조되는 행동선은 제한한다** (동시 강조 1~2개 권장 · 3개 이상은 설계 재검토).
- 피규어 장식은 상태/행동선을 가리지 않는다.
- **텍스트, 행동선, FX가 서로 같은 정보를 세 번 과하게 외치지 않는다** — 텍스트가 "무엇", 선이 "어디", FX가 "지금"을 나눠 맡는다.
- 위험도 높은 정보부터 우선순위를 둔다.
- **모바일 390px 기준에서 읽히지 않으면 실패다.**

**정보 우선순위** (높음 → 낮음):

1. 다음 보스 행동
2. 대상/피해 축
3. 사제가 수습할 수 있는 상태
4. 동료의 역할/실수
5. 분위기 장식

충돌 시 낮은 순위가 양보한다(장식이 먼저 죽고, 다음 행동이 마지막까지 남는다).
