## 1. 기본 사슬 (✅ 문법의 뼈대)

```
피해 예고 → 선대응 → 사고 발생 → 정화/보호/치유/예약힐 선택 → 동료 기회 발동 → 리포트 복기
```

| 단계 | 뜻 | 현재 구현 근거 |
|---|---|---|
| ① 피해 예고 | 큰 피해는 반드시 미리 말한다 | 경고바(다음 패턴) · 보스 시전바 · 광역 텔레그래프 ✅ |
| ② 선대응 | 예고를 읽은 자의 특권 | 보호막 · 소생 · 기도씨앗 · 선시전(+취소) ✅ |
| ③ 사고 발생 | 예고가 현실이 되는 순간 | 강타/심판 착탄 · 낙인/폭탄 부여 · 피해 플로트 ✅ |
| ④ 선택 | 답은 있으나 손은 하나 | 6슬롯 · GCD · 마나가 배타성을 강제 ✅ |
| ⑤ 동료 기회 | 사제 개입의 보상 | **정화 → 도적 차단 성공** ✅ (원형·유일 구현) |
| ⑥ 리포트 복기 | 거울 | 리포트 + ins 인사이트 ✅ |

## 2. 문법 규칙 (✅ 잠금)

1. **예고 없는 대형 피해 금지.** ⛔ 기습은 소형(잔피해)까지만. 대형기는 시전바·경고바·텔레그래프 중 최소 1개로 예고된다.
2. **모든 사고는 답 있는 질문이다.** 현재 로드아웃 풀 안에 대응 수단이 최소 1개 존재해야 한다(장착 여부는 플레이어의 선택·책임).
3. **선택은 배타적이다.** GCD 1.0s·마나·시전 시간이 "두 답을 동시에" 못 하게 만든다 — 기회비용이 문법의 심장 박동.
4. **동료 기회는 사제 개입으로만 열린다.** 동료가 스스로 사고를 해결하기 시작하면 문법 붕괴(동료 힐러화 ⛔의 문법적 표현).
5. **복기는 문장으로.** 숫자 나열이 아니라 다음 판의 행동을 바꾸는 말(거울 원칙).
6. **결정론.** 같은 판은 같은 질문을 한다 — 변수는 오직 사제의 손. ⛔ RNG 패턴.

## 3. Visual Battle Language (행동선) ★핵심

**정의: 행동선은 장식이 아니라 "누가 누구에게 무엇을 하려는지"를 보여주는 전투 언어다.** 선이 정보를 갖지 않으면 그리지 않는다. ⛔ 장식선.

### 3-1. 선 어휘 (현재 구현 상태 포함)

| 선 | 문장 | 구현 |
|---|---|---|
| **단일 대상선** (보스→아군) | "다음 강타는 너다" — 강타 시전 중에만 표시 | ✅ 02B `fxBossLine` |
| **광역 텔레그래프** (보스→파티) | "전원이 맞는다" — 심판 시전 중 전장 틴트+데칼 | ✅ 02B `fxAoe/fxAoeDecal` |
| **차단선** (도적→보스) | "내가 끊을 수 있다" — 차단 가능 심판 중 표시. **도적이 낙인이면 선이 끊어져(broken) 보인다** — "정화해야 이 선이 산다"가 화면에 그려지는 문법의 백미 | ✅ 02B `fxIntLine(+broken)` |
| **개입선** (사제→아군) | "지금 너에게 간다" — 스킬 사용 순간 0.46s, 종류별 색(치유/보호/해제/구원) | ✅ 02B `fxPriLine` |
| **피규어 링** | 표적/광역/선택 상태 표기 | ✅ 02B `figRing` |
| 사제 주시선 (보스→사제) | "다음은 리베로다" | ⏸️ **게이트 종속** — Core Gate Decision 후 |
| 씨앗 전이 흔적 | 씨앗의 여행 경로 잔상 | 🔷 (미세 연출 — 정보 예산 검토 후) |

### 3-2. 동시성 규칙 (✅)

- **화면에는 핵심선 1~2개만.** 현재 코어의 "보스 시전 동시 1개" 모델이 이를 **구조적으로 보장**한다(강타선과 광역 틴트는 동시 불가, 차단선은 광역 시전 중에만). 개입선은 0.46초 순간 표시라 잔상을 만들지 않는다.
- 미래 보스 셸도 이 규칙을 상속한다 — **동시 다중 시전 ⛔(1차 범위).**
- 충돌 시 선 우선순위: **차단선 > 단일 대상선 > 개입선 > 링.** 광역 틴트는 배경 레이어로 공존 허용.

### 3-3. 예고 언어의 계층 (17 문서와 공유)

멀리서 가까이로: **경고바(수 초 뒤 — 다음 패턴) → 보스 시전바(지금 진행 — [차단 불가] 명기) → 행동선/텔레그래프(공간) → 디버프 칩(개인 상태).** 같은 정보를 두 층 이상에서 중복 절규하지 않는다 🔷(정보 예산 원칙).

## 4. 위협 배지 실데이터화 — 유키 승인 1단계 (🔷 어댑터 전용)

현재 배지(안정/상승/위험/표적)는 02A의 시각 규칙(placeholder). 승인된 1단계는 **어댑터 lookahead**로 실데이터화:

| 배지 | 실데이터 규칙(초안) |
|---|---|
| 표적 | 현재 근접 표적(`mt`) — 이미 실데이터 ✅ |
| 위험 | 강타 시전 대상 **또는** 위험 디버프(낙인/폭탄) 보유 |
| 상승 | 스크립트 예측: **향후 ~6초 내** 이 대상에 위협 이벤트 예정 (lookahead) |
| 안정 | 위 해당 없음 |

- 구현 원칙: **코어 무접촉**(SCRIPT 읽기 전용) · 6초 창은 손맛 값(나라 폰 실기로 튜닝) · 배지 문구/색은 기존 유지.
- 소실 본체(⏸️)는 이 배지가 만드는 "읽기 문화" 위에 Core Gate Decision 이후 얹는다.

## 5. 금지 (⛔)

예고 없는 대형 피해 · 답 없는 사고 · 동시 3선 이상 · 정보 없는 장식선 · RNG 패턴 · 게이트 결정 전 코어 표적/SCRIPT 접촉 · 동료의 자가 사고 해결(힐러화).
