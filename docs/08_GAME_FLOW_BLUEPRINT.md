## 1. 핵심 루프 (한 바퀴)

1. **마을에서 준비한다.**
2. **성당에서 사제 스킬을 고르고 저장한다.**
3. **길드/게시판에서 싸울 몬스터를 고른다.**
4. **전투 전 위험 태그와 추천 준비를 본다.**
5. **전투에서 레이드프레임을 읽고 수습한다.**
6. **결과 리포트에서 내가 무엇을 잘했는지/놓쳤는지 확인한다.**
7. **보상/강화/해금을 통해 다음 전투 준비가 바뀐다.**
8. **마을로 돌아온다.**

루프의 감정 곡선: **준비(답을 고른다) → 예고(문제를 읽는다) → 수습(리베로) → 거울(다음 판이 바뀐다)**. 어느 단계도 다른 단계를 건너뛰게 만들면 안 된다 — 특히 7번(보상)이 5번(수습의 재미)을 대체하는 순간 성소판이 아니게 된다.

## 2. 현재 임시 흐름(07) vs 장기 흐름

**현재(07, 임시)**
```
타이틀 → 마을 → 성당(편집 + [전투 시작]) ────────┐
              → 길드(모르가스 [토벌]) ──────────┤→ 전투 → 리포트 → 마을
```
- 성당의 **[전투 시작] 버튼은 임시**다(07 카드에서 명시적으로 임시 채택). 검증용 지름길이지 최종 동선이 아니다.
- 로드아웃은 세션 유지·새로고침 시 기본 복귀·localStorage 미사용(07 사양).

**장기(목표)**
```
타이틀 → 마을 ─ 성당: 고르고 저장 → 마을로 복귀
             ─ 여관/동료소: 동료 확인
             ─ 길드 보드: 몬스터 선택(위험 태그·보상)
                  → 출정 확인(로드아웃+동료+태그 최종 점검)
                       → 전투 → 리포트 → 보상/해금 → 마을
```
- 원칙: **성당 = 답을 저장하는 곳, 길드 = 문제를 고르는 곳, 출정 확인 = 답과 문제를 맞대보는 곳.**
- 전투 진입은 장기적으로 **길드 보드 경유로 단일화**한다. (성당 시작 버튼 제거는 08A/08B 시점)

## 4. 전투 전 정보 설계 — 위험 태그 시스템

원칙: 몬스터 소개는 스탯 자랑이 아니라 **"이 몬스터가 어떤 사고를 만드는가"**의 예고다. 플레이어는 태그를 읽고 성당 로드아웃으로 **답을 준비**할 수 있어야 한다. (조사노트00의 교훈: 기억에 남는 기믹은 힐량이 아니라 **힐러의 행동을 제약**한다 — 태그는 그 제약의 예고편이다.)

**태그 어휘 후보 (v1, 8종)** — 🔷
| 태그 | 뜻 | 로드아웃이 주는 답 (예) |
|---|---|---|
| ⚡ 차단 필요 | 차단 가능한 대형 시전 | 정화(낙인 해제→도적 차단 연계) |
| 🩸 낙인 | 정화 필요 디버프 | 정화 |
| 💥 시한 폭발 | N초 내 정화 못 하면 폭발 | 정화 + 손 빠르기 |
| 🌊 광역 피해 | 파티 전체 동시 피해 | 공명의 기도 / 기도씨앗 |
| 🔨 탱커 폭사 | 탱커 대상 대형 타격 | 보호막 / 깊은 치유 / (후보) 수호 서약 |
| 🕯️ 지속 피해 | 잔상처 누적 | 소생 |
| 👁️ 사제 주시 위험 | 사제가 표적이 되는 순간 | (후보) 소실 / 결속 치유 |
| ⏳ 마나 압박 | 장기전·자원 소모전 | (후보) 성심 집중 / 성소의 호흡 |

- **현재 보스로 검증**: 모르가스는 이미 `guild` 화면에 태그 원형을 갖고 있다 — `⚡ 어둠의 심판 (차단)` / `🩸 낙인 · 💥 불안정한 마력 (정화)` / `🔨 탱커 강타` / `🔥 60초 광폭화`. 즉 **위험 태그 시스템은 신규 발명이 아니라 기존 씨앗의 승격**이다.
- **로드아웃 대조 경고** 🔷: 태그가 요구하는 답이 로드아웃에 없으면 출정 확인에서 경고. **경고만, 잠금 아님** — 07 정화 경고에서 유키가 확정한 "후자안(경고만)" 철학을 계승한다. 일부러 오답 로드아웃으로 도전하는 자유는 플레이어의 것.
- 태그는 **보스당 2~4개**로 제한(전부 위험하면 아무것도 위험하지 않다).

## 5. 결과 리포트 설계 — "다음 판을 위한 거울"

**현재 노출(✅)**: 총 회복(+오버힐%) · `· 소생 N`/`· 기도씨앗 N` 소항목(06) · 보호막 흡수 · 정화 횟수(폭탄 무효 포함) · 차단 성공/실패 · 사망자 · 특이사항 인사이트(`ins`).

**코어가 이미 집계 중인데 아직 노출 안 된 값(🔷 — 노출만 하면 됨, 저비용)**: 마나 바닥 시점(`oomT`) · 최저 마나(`manaMin`) · 신성력 피크(`holyPeak`) · 전투 의지 활용 시전 수(`valorCasts`) · 탱커 저체력 노출 시간(`tankLowMax`).

**확장 원칙**
1. 숫자는 나열이 아니라 **문장으로 번역**한다 — "불안정한 마력 2회가 폭발했다. 3초 안의 정화는 피해 자체를 지운다"(현재 `ins` 스타일 유지·강화).
2. 오버힐은 **낭비의 지표이자 예측 실패의 힌트**로 해석해 준다(조사노트00: 유효 치유량 문화의 번역). 단, 오버힐이 미덕인 예외 보스(스루풋 체크형)를 만들면 해석 문장도 반전시킨다.
3. **다음 판 힌트** 🔷: 리포트 마지막 줄은 "다음 판에 하나만 바꾼다면?" — 로드아웃 제안까지 이어지면 루프 7번(준비가 바뀐다)과 접속된다.
4. 스킬별 회복 분해 정밀화(코어 승격)는 선택 과제 — 어댑터 ledger(06)로 충분한 동안은 보류.
