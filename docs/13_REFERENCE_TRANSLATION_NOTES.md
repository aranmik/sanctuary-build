# 성소판 13 — 레퍼런스 번역 노트 (REFERENCE TRANSLATION NOTES)
**작성: 렌 · 2026-07-07 · EP08 설계팩 6/6 · 상태: 렌 초안(유키 판정으로 확정)**

> **성소판은 5인 파티의 전장 상황을 읽고 수습하는 사제 게임이다. 플레이어는 공격하지 않는다. 플레이어는 파티의 리베로다.**
> 조사노트 00~04(WoW 클래식~TBC 힐러 조사)는 성소판의 **원형 레퍼런스**다. 성소판은 독립 게임이며, 원형에서 가져오는 것은 시스템이 아니라 **"왜 재미있었는가 / 어떤 판단을 만들었는가 / 어떤 역할 경계를 만들었는가"**다.

## 0. 사용 원칙 (✅ 잠금)

1. 조사노트의 스킬명/직업명/시스템명은 **originRef로만** 사용한다.
2. 실제 성소판 표시명/시스템명은 **성소판 고유명**으로 번역한다.
3. **수치/공식/스킬 구조를 그대로 가져오지 않는다.**
4. 추출 대상은 재미의 원리 — 판단·감정·역할 경계.
5. **동료가 진짜 힐러가 되는 방향은 ⛔.**
6. **성소판의 힐러는 오직 플레이어 사제뿐이다.**

## 1. 조사노트 00 (시대배경·힐러메타) → 성소판 번역

| 원형의 원리 | 성소판 착지점 | 상태 |
|---|---|---|
| 마나 압박 — "HPS가 아니라 마나가 보스보다 오래 버티는가" | 마나 운영이 이미 판정 축(리포트 `oomT`/`manaMin` 집계 중). ⏳ 마나 압박 태그 보스, 자원 운영 스킬(성심 집중·성소의 호흡)로 확장 | ✅ 씨앗 / 🔷 확장 |
| 오버힐 = 순수 낭비 → 유효 치유량 문화 | 리포트가 오버힐 %를 거울로 노출 — "낭비의 지표이자 예측 실패의 힌트" | ✅ |
| 선시전 후 캔슬(스톱캐스팅) | **시전바 클릭 취소 이미 구현** — 예측 플레이의 문이 열려 있음 | ✅ |

| 디스펠 4타입 × 직업 매트릭스 = 편성 강제 | 1힐러 게임 번역: 편성이 아니라 **로드아웃 선택**으로 — 디버프 타입별 대응 스킬(정화/대정화/시한 폭탄) | 🔷 |
| 힐 배분 문화(어싸인) | 1인 번역: **출정 전 로드아웃 = 나 자신에게 내리는 배분표** — 출정 확인 화면의 정서적 원형 | 🔷 |

| 40→25인 = 개인 기여 해상도 | 성소판은 극한값 — **힐러 1인, 기여 해상도 100%.** 정의 자체가 이 시사점의 완성형 | ✅ (정의) |
| 러스트류 결전 창(체인 러스트) | **전투 의지(valor) 창이 이미 코어에 존재** — 스크립트가 여는 가속 창을 사제가 활용(`valorCasts` 집계) | ✅ |

## 3. 조사노트 02 (성기사) → 역할 경계 참고

- **가져올 원리**: ①단일 대상 전문성이라는 "역할의 뾰족함"(스페셜리스트의 매력) ②외생기 감각 — 신의 축복(LoH)류 "최후 버튼"의 무게(긴급 구원과 공명) ③**인내(Forbearance) — 강력한 효과들이 제약을 공유하는 설계 패턴**(보호막 잠금 6s가 이미 같은 혈통) ④축복 = 전투 전 준비 의식 → 성물/패시브의 정서적 원형.
- **⛔ 가져오면 안 되는 것**: **빛의 심판(아군이 때리면 자동 회복) — 동료 힐러화 위험 소재 1호.** 탱힐 머신의 동료화, 축복 관리 노동(PallyPower식 복잡성), 인원수만큼 버프가 느는 구조.
- 성기사가 미래 동료 후보가 되더라도 10 문서 §4의 상한("사제 없으면 못 버티는 정도")을 잠근다.

## 4. 조사노트 03 (주술사) → 보조 감각 참고

- **가져올 원리**: ①**토템 = 설치형 선택, 슬롯 경합이 곧 의사결정**("물 슬롯에 뭘 둘까") → 주술사 동료의 출정 준비 선택지(10 §5) ②러스트 = 결전 창(코어 전투 의지로 이미 번역됨 ✅) ③진동/접지 토템 = 위험 예고·1회 무효라는 "보조의 문법" ④토템은 부서진다 — 보조에도 리스크가 있다는 감각.
- **⛔ 가져오면 안 되는 것**: **치유의 샘 토템·연쇄 치유·대지의 보호막 — 자동/스마트 치유는 전부 힐러화 직행.** 주술사 동료의 완충은 "사제의 일을 돕는" 수준 상한(10 §2).

## 5. 조사노트 04 (드루이드) → 예약 치유 감각 참고

- **가져올 원리**: ①HoT = 예방적 치유·미래에 맡기기 — **소생으로 이미 번역·실기 검증 완료**("공대 힐러 느낌" PASS의 원형) ②롤링/갱신의 긴장 — 유지 관리가 만드는 리듬(소생 재시전 판단) ③만개(Bloom) = 예약 폭발 — 기도씨앗의 발동 감각과 공명 ④정신 자극 = 자원 기부 → 성물/유물 후보 감각 ⑤나무 껍질 = 자기 생존기 → 사제 생존 축 후보.
- **⛔/⏸️ 경계**: **전투 부활(환생) — 동료가 갖는 것은 ⛔(힐러화+사제 고유 결단 침식).** `docs/01`의 "아군 죽으면 수습 실패(전투 중 부활 1차 제외)" 각주가 열어둔 1회 예외는 **사제의 무거운 결단 스킬로만** 검토 가능하며, 1차 범위에서는 ⏸️(유키 판정 필요). 변신(생명의 나무)류 모드 전환도 1차 범위 밖.

## 6. 총괄 복제 금지 목록 (⛔)

1. WoW 스킬명/직업명 그대로 출시 UI 사용 (originRef는 문서 전용)
2. 25인/40인 레이드 운영 구조 복제 (배분표·버프 노동·소모품 경제)
3. **동료 힐러화 — 어떤 경로로도** (자동 치유·스마트 힐·전투 부활 포함)
4. 복잡한 직업 편성 매트릭스 (성소판은 로드아웃으로 답한다)
5. 수치/공식 복제 (계수·랭크·마나 공식)
6. **확률 기반 발동** (성소판 결정론 침해 — 확률은 확정 조건으로 재번역)
7. 시스템 여백 없이 노동만 남는 준비(숙제화)
