# 34. 보스별 결과 거울 문장 (BOSS-SPECIFIC REPORT HINT)

**작성: 렌 · 2026-07-09 · EP22 · 상태: 구현 카드(PASS 후보 — 나라님 폰 실기 대기) · 패치 p17 · 어댑터 전용(CORE 무접촉)**
**상호 링크**: [33 보스 문법](./33_BOSS_GRAMMAR.md)(§8 실패 3층 · 본 카드의 근거) · [27 데모 정의](./27_FIRST_DEMO_COMPLETION_DEFINITION.md) · [31 파쇄자](./31_IRON_CRUSHER_RUNTIME.md) · [32 심연](./32_THIRST_ABYSS_RUNTIME.md) · [29 출정 경고](./29_MATRIX_SORTIE_WARNING_CONTRACT.md)(전투 전 거울 — 본 카드는 전투 후 거울)

> **핵심 문장 3줄**
> 1. 결과 화면의 힌트를 **보스별로** 만든다 — 이번 보스가 던진 질문(판단/화력/인내)을 기준으로 "다음에 무엇을 바꿀지"를 비춘다.
> 2. 꾸짖는 문장이 아니라 **다음 준비로 이어지는 거울**이다. 승리엔 답을 확인해주고, 패배엔 어느 층(파티/스킬/판단)을 볼지 가리킨다.
> 3. 코어 무접촉 · 기존 `.ins gold` 슬롯 재사용(신규 UI 0) · 공용 힌트는 폴백으로 존치.

---

## 1. 목적

기존 결과 화면은 공용 힌트 1줄(💡)을 보여줬다 — 모르가스 어휘(낙인·정화·차단)에 맞춰진 문장이라 파쇄자·심연에서는 결이 어긋났다. 본 카드는 **현재 보스가 무엇을 물었는지**를 기준으로 결과 거울을 보스별로 정확하게 만든다. 새 보스도, 밸런스 변경도 아니다 — **거울의 해상도**를 올리는 카드다.

## 2. Boss Grammar 01과의 연결

[33 §8]의 **실패 원인 3층 구조**(①파티 구성 ②스킬 구성 ③전투 중 판단)를 결과 화면의 언어로 옮긴다. 각 보스의 거울은:
- **승리 시**: 그 보스의 질문에 어떻게 답했는지 확인해준다(핵심 선언문의 "다시 세운다"의 성취).
- **패배 시**: 1줄은 이번 판에 흔들린 지점, 2줄째는 **다음 판에 확인할 층**(스킬 구성 + 전투 판단)을 자연스러운 문장으로 가리킨다. 새 UI 태그 없이 문장에 녹였다([33 §8] "문장에 자연스럽게 녹여도 된다").
- [33 §6] 리베로 원칙: "사제는 만능이 아니다 — 전투 전 준비가 수습 가능성의 범위를 정한다"를 거울이 상기시킨다(2줄째의 "성당에서 챙겼는지" 계열).

## 3. 보스별 힌트 원칙

- **bossId(CUR_BOSS) 분기 + 결과 스탯(r) 판독** → 보스별 함수가 1~2개 문장 반환. **최대 2줄**.
- 승리=1줄(답 확인) · 패배=2줄(흔들린 지점 + 다음 층). 출정 차단·전투 결과·승패 판정·수치 **일절 무변경**.
- 어휘 격리(각 보스 정체성을 흐리지 않음 — [33 §9] 금지행 준수):
  - 모르가스 거울엔 정화/차단/낙인(판단) 어휘만, 강타/탱커 어휘 ⛔.
  - 파쇄자 거울엔 전사/강타/보호막(화력·버티기) 어휘만, 정화/차단 어휘 ⛔.
  - 심연 거울엔 마나/소생/성심 집중(인내) 어휘만, 보호막/정화 어휘 ⛔.
- **공용 힌트 존치**: 미인식 보스이거나 보스 함수가 빈 결과면 기존 공용 힌트로 폴백 — 공용 로직은 죽지 않는다.

## 4. 모르가스 힌트 (판단형)

| 조건 | 문장 |
|---|---|
| 승리 | 정화와 차단 연결이 전장을 살렸습니다. |
| 차단 실패(낙인 연루) | 차단이 흔들렸다면 정화 타이밍과 긴급 수습 스킬을 같이 보세요. |
| 그 외 패배 | 낙인을 정화해야 도적의 차단 기회가 살아납니다. |
| 패배 2줄째(층) | 다음 판: 성당의 정화, 도적의 차단 타이밍, 그 사이 수습을 함께 보세요. |

*주의*: 모르가스를 단순 딜/탱커 보스로 해석하지 않는다.

## 5. 강철의 파쇄자 힌트 (원초 레이드 체크형)

| 조건 | 문장 |
|---|---|
| 승리 | 전사는 버티고, 사제는 살리고, 딜러는 고통을 끝냈습니다. |
| 전사 사망 | 전사가 무너지면 전장이 무너집니다. 강타 타이밍을 먼저 보세요. |
| 보호막 흡수 낮음(<800) | 강타 전에 전사에게 보호막을 맞추면 전선이 오래 버팁니다. |
| 그 외 패배 | 강타를 보호막으로 받아냈습니다. 마나가 버틴다면 처치까지 노려보세요. |
| 패배 2줄째(층) | 다음 판: 보호막을 쥐고 갔는지, 강타 예고에 미리 걸었는지 확인해보세요. |

*주의*: 기본형 파쇄자에 어그로 사고/광역/차단을 섞지 않는다(변주 보스 구현 아님).

## 6. 갈증의 심연 힌트 (자원·인내형)

| 조건 | 문장 |
|---|---|
| 승리 | 마나가 말라도 끝까지 버틴 것이 승리의 답이었습니다. |
| 마나 조기 고갈(oomT<40) | 마나가 일찍 바닥났습니다. 초반 큰 치유를 줄이고 소생·성심 집중 타이밍을 보세요. |
| 오버힐 높음(≥20%) | 넘치는 치유는 심연에서 곧 마나 손실이 됩니다. 딱 필요한 만큼만. |
| 그 외 패배 | 마나가 마르기 전에, 무엇을 안 쓸지 먼저 정해보세요. |
| 패배 2줄째(층) | 다음 판: 소생·기도씨앗·성심 집중을 챙겼는지, 큰 치유를 아꼈는지 확인해보세요. |

*주의*: 심연을 정화/차단/보호막 체크 보스로 흐리지 않는다.

## 7. 실패 원인 3층 구조와의 연결

- **2층(스킬 구성)** · **3층(전투 판단)**은 각 보스 패배 2줄째가 담당(성당에서 무엇을 챙겼나 + 그 순간 무엇을 골랐나).
- **1층(파티 구성)**은 현재 4동료 고정이라 문장에 직접 얹지 않았다 — 편성 시대에 열린다([33 §8]).
- **심화 4~5층(동료 특성/스킬 강화)**은 문서 전용 · 본 카드 구현 ⛔.

## 8. 구현 범위 (p17 · 2편집 · 어댑터 전용)

- e1: `pickReportHint` 뒤에 `BOSS_HINT_FN`(보스별 함수 3종) + `bossHintLines(bossId,r)` + `reportMirror(r)` 삽입. r 스탯 + 어댑터 전역 `CUR_BOSS`만 읽는다 — **코어 무접촉**(report()·CFG·SCRIPT 무변).
- e2: `showReport`의 힌트 슬롯 — 공용 1줄 → `reportMirror(r)`(보스별 최대 2줄). 기존 `.ins gold` 슬롯 재사용, **신규 UI/CSS 0**.
- 공용 `pickReportHint`는 `reportMirror` 폴백으로 존치(미인식 보스·빈 결과). 기존 코어 인사이트(`r.ins`)는 그대로 위에 렌더 — 완전 보존.
- 기준선: index.html **117,251 B / 1,920줄 / md5 `9829b5b0b9c344cadbb3c5c0d1134d79`** (이전 114,688/1,882/e5c7ca06 — 어댑터 +38줄). **CORE 466줄/22,521 B 불변**(byte-identical) · 3보스 스모크 51.4/48.5/61.8 유지. baseline_check는 전체 파일 숫자만 갱신(CORE·스모크 무변).
