# 성소판 15 — 몬스터/보스 청사진 (MONSTER BOSS BLUEPRINT)
**작성: 렌 · 2026-07-07 · EP09 설계팩 2차 2/8 · 최근 동기화: EP17B(데모 라인업 확정) · 상태: 렌 초안(유키 판정으로 확정)**

> **성소판은 5인 파티의 전장 상황을 읽고 수습하는 사제 게임이다. 플레이어는 공격하지 않는다. 플레이어는 파티의 리베로다.**
> **보스는 스탯 덩어리가 아니라 "사제에게 질문을 던지는 장치"다.** 좋은 보스 = 좋은 질문. 모든 보스는 [14 전투 문법](./14_COMBAT_GRAMMAR_BLUEPRINT.md)을 준수한다.

- 관련: [08 플로우 §4 위험 태그](./08_GAME_FLOW_BLUEPRINT.md) · [09 스킬](./09_PRIEST_SKILL_BLUEPRINT.md) · [13 번역 노트](./13_REFERENCE_TRANSLATION_NOTES.md)
- **이번 카드에서 신규 보스 구현 ⛔ — 아래는 전부 설계 셸이다.** 태그: ✅확정 / 🔷후보 / ⏸️보류 / ⛔금지

## 1. 보스 설계 형식 (✅ 7필드 잠금)

모든 보스(현재·미래)는 다음 형식으로 기술한다:
**목적(질문) / 위험 태그 / 요구 로드아웃 / 주요 사고 / 동료 기회 / 실패 리포트 문장 / 구현 상태**

- "실패 리포트 문장"은 이 보스의 서명이다 — 패배가 모욕이 아니라 **다음 판의 열쇠**가 되도록 쓴다(거울 원칙의 보스 확장).

## 3+. 보스와 행동선 (Visual Battle Language 준수 · ✅ 잠금)

모든 보스는 자신의 패턴을 **행동선 어휘(14 §3)로 말할 수 있어야 한다** — 단일 대상 패턴은 단일 대상선으로, 광역은 텔레그래프로, 차단 가능 시전은 차단선으로. 어휘에 없는 패턴을 도입하려면 선 어휘 확장(14 개정)을 먼저 거친다. **동시 핵심선 1~2개 규칙은 보스 설계의 상한**이다 — 동시 다중 시전 ⛔(1차 범위). 위 셸 5종은 전부 기존 선 어휘로 표현 가능함을 확인했다(3-E 주시형만 신규 "사제 주시선" 필요 → 게이트 종속과 일치).

## 4. 태그 ↔ 요구 스킬 ↔ 리포트 연결 (✅ 원칙)

**위험 태그(질문 예고) → 요구 로드아웃(답 준비) → 사고(질문 실행) → 리포트 문장(채점과 열쇠).** 이 네 고리가 끊긴 보스는 출고 불가 — Boss Grammar Conformance 체크 항목.
