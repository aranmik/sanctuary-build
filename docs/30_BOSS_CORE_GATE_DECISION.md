## 3. A안 — 최소 분기형의 의미 (✅ 문장 확정 · 일부 実測 이월)

현재 코어 구조를 실측(28 §1~§2 · 본 빌드 재확인)한 위에서, "최소 분기형"을 다음 문장으로 확정한다.

**A안이 뜻하는 것:**
1. **Morgas 전투의 손맛과 51.4 물리 법칙을 보존한다.** `createGame()`은 **무인자 호출 시 모르가스로 고정**(현재 638행 · 무인자) → 스모크/하네스 호출부(`G=createGame()` 1039행 · `__seedHealer.smoke` 하네스)가 **무수정으로 하위호환**된다.
2. **기존 모르가스 SCRIPT / 전투 흐름을 깨지 않는다.** 현 `SCRIPT`(619행 · 17이벤트 고정 타임라인)는 모르가스 정의로 그대로 보존한다.
3. **보스별 데이터 분기는 최소한으로 추가한다.** 예상 형태(28 §4 A안): 코어 내 `BOSS_DEFS={morgas:{boss:<현 CFG.boss>,script:<현 SCRIPT>}, crusher:{…}}` 색인 + `createGame(bossId='morgas')`에서 `B/SC` 룩업. **★최종 스키마명·형태는 다음 구현 카드에서 실측 확정.**
4. **전투 엔진 전체 재작성은 금지한다.** `fireScript`/`resolveBossCast`/스케줄러(867행 루프) 등 사고 로직은 데이터 구동으로 "완전 데이터화"하지 않는다 — 이는 B안의 영역이며 본 게이트는 A안이다.
5. **새 보스 실전투는 기존 loop / adapter / UI를 최대한 재사용한다.** 어댑터 전선(`enterBattle(slot)` 1246행 — 현재 slot 파라미터 미사용 · `newGame` 1037 · `sortieGo` 1185)은 이미 문턱까지 와 있어, 연결은 **3줄 내외**로 예상된다(28 §2-②). 예보(`nextThreatFor` 1565행 · 전역 `SCRIPT` 직독 1567)는 반환 객체에 `script` 1필드를 더해 재배선한다.

**코어 편집 표면적(28 §2 표 기준 추정)**: 십수 줄 — ①보스 선택값 수령(createGame 인자화) ③SCRIPT 분기(BOSS_DEFS 색인 + 룩업) ④예보 재배선(반환 +1필드) ⑤이름 보간(코어 리터럴 7곳 → `B.name`). ★ **정확한 편집 위치·줄 수는 다음 구현 카드(Iron Crusher Runtime 01)에서 코어를 실제로 열며 실측**한다. 본 결정서는 "이 표면적 안에서만 연다"는 **상한**을 정하는 문서다.

## 6. 강철의 파쇄자 착공 전제 (✅ — 다음 카드가 검증할 질문)

- 강철의 파쇄자는 **강타 / 버티기 질문**이다. 질문 문장(15·빌드 1:1): **"네 마나는 나의 파괴보다 오래 버티는가?"**
- 정화 / 차단 보스가 **아니다** — **보호막 / 선(先)대응 / 버티기 운영**을 묻는다. (모르가스와 손맛이 "서로 다르게" 느껴져야 데모가 성립 — 27 §1.)
- **「보호막 없음」은 위험으로 유지한다.** (29 §4-2 · Sortie Warning 01에서 이미 danger로 데이터화되어 있음 — 실전투가 이 경고를 "사실"로 만든다.)
- **전사 / 주술사 등 기존 4동료만 사용한다.** 신규 동료 추가 없이 성립해야 한다.
- 어휘: 기존 6종(`smash`/`brand`/`judge`/`bomb`/`valor`/`enrage`)의 재조합으로 강타/버티기형 타임라인을 구성 — **어휘 무확장이 목표**(28 §4 A안 저위험 근거).
- 15 §4 "네 고리" 성립 + 표준 검증 루틴 + 폰 실기가 PASS 조건. ★ 구체 타임라인·수치는 Iron Crusher Runtime 01에서 설계·실측.

- 갈증의 심연은 **마나 압박 / 절약 운영 질문**이다. 질문 문장(15·빌드 1:1): **"무엇을 안 쓸 것인가?"**
