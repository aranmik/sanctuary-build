## 5. 무대 언어가 장식과 다른 이유

- 장식 FX는 "예쁘다"에서 끝난다. 무대 언어는 **읽힌다** — 각 표현이 고정된 의미를 지고, 그 의미가 실제 전투 계산과 일치한다. 선 하나가 "누가 누구에게"를 말하고, 데칼 하나가 "어느 공간이 위험한지"를 말하며, 임펄스 하나가 "막혔는지 뚫렸는지"를 말한다. 의미 없는 선은 그리지 않는다. → **표현의 경제**가 세련됨의 핵심.

## 6. Actor Pose — 배우의 몸짓 (5-1)

- **담당 의미**: 현재 무엇을 준비/수행/방금 겪었는가, 역할상 어떤 태도인가.
- 예: 전사 방패 앞세움 / 도적 차단 준비 / 마법사 충전 / 주술사 유지 집중 / 파쇄자 강타 준비.
- **원칙**: 포즈는 관계선을 **대신하지 않는다**. 포즈만으로 "지금 무엇을 하는가"를 말하고, "누구와 연결되는가"는 필요할 때 행동선이 말한다. 포즈는 [51] Pose Pack 계승, 이 계약은 그 위에 관계·결과 층만 얹는다.

## 7. Tell Decal — 예고 데칼 (5-2)

- **담당 의미**: 곧 위험해질 **공간**, 공격이 점유/휩쓸 영역, 압력의 방향, 바닥·공간에 남는 예정된 사건.
- 예: 파쇄자 망치 낙하 압력 영역 / 모르가스 저주가 비틀 연결 영역 / 심연 흡수 웅덩이.
- **원칙**: 데칼은 "이 스킬을 누르세요"를 말하지 **않는다**. 위험의 공간과 성격만 보여주고 정답을 지시하지 않는다. 기존 경고 텍스트가 이미 말한 내용을 큰 데칼로 **중복하지 않는다**(§17).

## 8. Action Line — 행동선 (5-3)

- **담당 의미**: source Actor와 target Actor 사이의 **관계**, 공격·차단·보호·흡수·유지의 방향성, 실제로 누가 누구에게 작용하는가.
- **원칙**: 행동선은 **화면 중심/모서리에서 시작하지 않는다**. 항상 의미 있는 **anchor에서 출발·도착**(§13). 예: 망치 머리 → 전사 방패층 / 도적 단검 → 보스 시전 anchor / 마법사 구체 → 대상 anchor / 심연 흡수구 → 대상 생명 anchor. [41 §I] "선은 버튼을 가리키지 않는다·전장 객체 사이에서 발생한다" 계승.

## 9. Aura / Field — 지속 상태 (5-4)

- **담당 의미**: 계속 유지되는 역할/영향(도발·보호·지원·주술 유지), Actor 주변에 머무는 상태의 분위기.
- **원칙**: 오라는 행동선처럼 **목표를 가리키지 않는다**. 지속 상태를 말하되 화면을 덮거나 모든 Actor를 연결하는 장식이 되면 안 된다. 기존 sb-aura(전사 파랑·주술사 청록 발광)가 선례.

## 10. Contact / Impulse — 충돌과 순간 결과 (5-5)

- **담당 의미**: 실제로 닿은 순간, 막음, 차단 성공, 강타 적중, 보호막 파손, 정화 성공, 지원 파동 도달.
- **원칙**: **짧고 명확**해야 하며, **게임 계산 결과와 타이밍이 일치**해야 한다. 시각이 계산보다 앞서 결과를 확정하면 안 된다.

## 11. Afterglow — 결과의 잔광 (5-6)

- **담당 의미**: 방금 성공/실패한 사건의 짧은 기억, "무엇이 일어났는지"를 눈이 따라잡게 함, 수습·안정·해소의 여운.
- **원칙**: 잔광은 **다음 행동을 가리키지 않는다**. 결과를 확인시키고 **빠르게 사라진다**.

## 12. 사건의 6단계 (Intent → Resolve)

모든 사건이 6단계를 다 쓰지 않는다.

| 단계 | 표현 | 내용 |
|---|---|---|
| 1 Intent 의도 | Actor 포즈 변화·미세 오라 | 공격/대응이 시작될 기미 |
| 2 Tell 예고 | 데칼·짧은 관계 예고 | 위험의 공간·방향이 읽힘 |
| 3 Commit 실행 확정 | 포즈 최명확·source anchor 활성 | 행동선 필요 시 이 시점부터 |
| 4 Travel 이동 | 공격/마법/차단 이동 구간 | 이동 없는 기술은 생략 |
| 5 Contact 접촉 | shield/body/cast anchor 충돌 | 가장 짧고 명확한 임펄스 |
| 6 Resolve 결과·회수 | 피해/차단/보호/정화 결과 | Actor recover/aftercast/sustain 복귀 + 짧은 잔광 후 종료 |

- **철칙**: 이 단계가 전투 **계산의 실제 타이밍을 왜곡하면 안 된다**. 시각이 계산보다 앞서 결과를 확정하거나, 이미 끝난 사건을 오래 끌면 안 된다.

## 13. Anchor 문법

`sb-` 네임스페이스 안, 행동이 실제로 발생하는 **파츠**를 연결한다(구현은 후속 카드).

- 제안 클래스: `.sb-anchor-source` `.sb-anchor-target` `.sb-anchor-cast` `.sb-anchor-shield` `.sb-anchor-body` `.sb-anchor-ground` `.sb-anchor-weapon` `.sb-anchor-orb` `.sb-anchor-totem` `.sb-anchor-party`.
- **원칙**: ①공격은 무기/공격 소품에서 출발 ②마법은 구체·손·토템 등 역할 소품에서 출발 ③보호막은 Actor 몸이 아니라 **별도 보호층에 도착** ④차단은 보스 몸 전체가 아니라 **시전 원점(cast anchor)**을 겨냥 ⑤바닥 범위는 Actor 중심점이 아니라 **ground anchor** ⑥파티 전체 지원은 개별 몸통을 억지로 선으로 묶기보다 **party/field anchor** 검토 ⑦**좌우 반전·scale이 변해도 anchor가 올바른 위치를 유지** ⑧**배치 transform과 anchor 좌표 계산의 책임을 섞지 않는다**.
- **핵심 문장**: `선은 피규어 중심점을 연결하지 않는다. 행동이 실제로 발생하는 파츠를 연결한다.`

## 14. Layer 문법 (하→상 우선순위)

1. **Stage Ground** — 바닥·그림자·낮은 데칼
2. **Behind-Actor Field** — 넓고 약한 분위기 오라(Actor 실루엣 안 가림)
3. **Actor Body** — 피규어 본체
4. **Protection Layer** — 방패·보호막·도발·수호층(몸보다 앞에서 공격을 받는 시각층)
5. **Relation / Action Layer** — source↔target 행동선(얼굴·핵심 장비 안 가리게 경로 조정)
6. **Contact / Impact Layer** — 충돌 순간 짧은 임펄스
7. **Result / Readability Layer** — 수치·상태·짧은 결과 피드백(기존 UI·renderFx 존중)

- **원칙**: 모든 효과를 Actor 위 최상단에 그리지 않는다. **방패가 막았다면 선이 몸까지 관통하지 않는다.** Actor 얼굴·방패 문장·단검·구체·토템 고리를 **지속적으로 가리지 않는다**. 카드 UI와 전장 무대의 FX 책임을 섞지 않는다. **기존 renderFx·target ring·상태 플래시를 제거/대체하지 않는다.**

## 15. 형태·질감 언어

색·굵기·형태를 직업색으로만 구분하지 않고 **행동 의미**와 함께.

| 문법 | 느낌 | 사용 후보 |
|---|---|---|
| **압박/강타** | 굵음·무거움·직선/짧은 각·느리게 응축·접촉 시 압축 | 파쇄자 강타, 무거운 보스 공격 |
| **차단/절단** | 얇고 빠름·은회/냉색 섬광·끊기는 박자·시전선을 가로지름 | 도적 차단, 시전 성공/실패 |
| **충전/투사** | 구체에서 응축·보라 본체+노랑 핵·구체 이동/방출 우선(직선 레이저보다)·발사 전후 포즈 연결 | 마법사, 투사형 마법 |
| **유지/지원** | 부드러운 호·원형/호흡 field·청록 고리/토템 중심·낮은 강도(화면 안 묶음) | 주술사 sustain, support pulse, 정화 |
| **저주/연결** | 비틀림·불안정한 실·중간 꼬임/분기·잘못된 연결 강조 | 모르가스 |
| **흡수/배수** | source→target 쏘는 선보다 **target→흡수구 빨림**·액체/리본/웅덩이·안쪽으로 말림 | 갈증의 심연 |

## 16. 동시 연출 혼잡 방지

모바일 390×844에서 **Actor와 장비가 먼저** 보여야 한다.

- 한 순간의 **주연 사건은 하나**. 다른 사건은 낮은 강도 조연으로 후퇴.
- 기본 primary relation 1개를 가장 명확하게 + 필요 시 supporting aura 또는 짧은 result impulse 1개 정도 보조.
- **수치 고정이 아니라 실제 화면에서 읽히는지 우선.**
- 여러 선이 교차하면 우선순위 낮은 선은 생략·약화. 얼굴·핵심 소품 가리는 선은 경로를 휘거나 중단.
- 경고 텍스트가 이미 말한 내용을 큰 데칼로 중복하지 않음. "모든 상태를 항상 표시" 하지 않음.
- **우선순위**: ①즉시 위험/보스 핵심 예고 → ②플레이어가 방금 만든 방어·차단 관계 → ③실제 충돌 결과 → ④지속 지원 상태 → ⑤장식성 잔광.

## 17. UI와 무대의 역할 분리

- **UI 담당**: 정확한 시간·수치·스킬명·상태 라벨·카드 선택·HP/MP·경고 텍스트.
- **무대 담당**: 누가 움직이는가·누구와 관계되는가·위험이 어느 공간에 생기는가·어디서 막히거나 끊겼는가·결과의 감정과 무게.
- **원칙**: `UI는 정확성을 말하고, 무대는 상황과 관계를 말한다.` 무대 FX 안에 긴 텍스트나 버튼 지시를 넣지 않는다.

## 18. 전사 문법

- 포즈: guard / brace / resolve. 핵심 anchor: shield / body / weapon.
- 언어: **보호층·저지·받아냄**. 선보다 방패와 충돌층이 우선. 보호막이 있으면 공격은 shield anchor(보호층)에서 멈추고 몸을 관통하지 않는다.

## 19. 도적 문법

- 포즈: interrupt ready / dash / interrupt hit / recover. 핵심 anchor: front dagger / cast target.
- 언어: **짧고 빠른 절단·차단**. 지속 선보다 순간적 끊김과 성공 임펄스 우선. 차단은 보스 몸/망치가 아니라 **시전 anchor**를 겨냥(§22-5).

## 20. 마법사 문법

- 포즈: charge / channel / release / aftercast. 핵심 anchor: orb / hand.
- 언어: **응축→방출→여운**. 보라 본체+노랑 핵 색 문법 유지([52]). 단순 레이저 직업으로 만들지 않는다 — 구체의 이동·방출이 우선.

## 21. 주술사 문법

- 포즈: sustain / cleanse ready / support pulse / rescue. 핵심 anchor: totem / ring / party field.
- 언어: **유지·정화·수습**. 모든 동료에게 항상 선을 연결하지 않는다. 원형 오라와 짧은 지원 파동을 우선 검토(party/field anchor).

## 22. 강철의 파쇄자 — 첫 구현 문법 (Iron Crusher Smash Line Rework 02가 따를 규격)

- **22-1 강타 준비**: 파쇄자 windup 포즈 + 망치 weapon/source anchor 활성 + 전사 guard/brace 연결 가능. 위험 공간은 낮은 강도 압력 데칼로 표시 가능. **아직 충돌선이 몸까지 꽂히지 않음.**
- **22-2 강타 실행**: 망치 anchor에서 전사 방향으로 압박 관계 확정. **화면 모서리→전사 직선 금지.** 반드시 망치 파츠에서 출발. 타깃이 전사면 shield anchor 우선, 보호막 없으면 body/target anchor로 전환.
- **22-3 보호막 보유 시** (§23): 선이 전사 몸을 **관통하지 않음** → 방패/보호층에서 멈춤. 보호층이 압축·진동·점등. 피해 감소/흡수 결과와 일치하는 임펄스. **전사 몸의 피격은 보호층 결과보다 앞서면 안 됨.**
- **22-4 보호막 없음** (§23): body anchor까지 접촉. 방패 자세는 유지 가능하나 보호층 차단 연출 없음. 무거운 피격 임펄스. 카드/HP 수치와 동기화.
- **22-5 차단 관계**: 도적 차단은 파쇄자 몸/망치가 아니라 **시전 anchor**를 겨냥. ★단 **현재 runtime에 실제 차단 가능 시점·이벤트가 무엇인지 후속 코드 검토 전 단정 금지**(§27).

## 23. 보호막 보유/미보유 분기

| 조건 | 도착 anchor | 몸 관통 | 임펄스 | 결과 동기화 |
|---|---|---|---|---|
| 보호막 보유 | shield / protection layer | ✕ (방패층에서 멈춤) | 보호층 압축·진동·점등 | 피해 감소/흡수와 일치·몸 피격은 보호층 뒤 |
| 보호막 없음 | body / target anchor | ○ (body까지 접촉) | 무거운 피격 | 카드/HP 수치와 동기화 |

- 이 분기는 기존 shell_iron 강타·보호막 상호작용([31])의 **시각 표현**이지 계산 변경이 아니다. 실제 shield 값과 block/absorb 결과에 맞춰 보호층의 압축·점등·파손 강도를 표현하며, **시각 연출이 실제 흡수량을 선행하거나 과장하지 않는다**. 정확한 수치(강타량·보호막량·흡수량)는 실제 코드 확인 전 단정하지 않는다(§27).

## 24. 모르가스 후속 문법 (방향만 · 미구현)

- 정체성: 꼬는 자·낙인·저주 실·연결의 오염·정상 관계를 비트는 존재.
- 문법: Actor↔Actor 꼬인 연결 / 보스 mask·cast anchor에서 시작하는 불안정 실 / 낙인 있으면 선이 흐려지거나 더 꼬임 / 정화·차단 성공 시 연결이 풀리거나 절단 / 단순 빨간 공격선보다 **비틀린 관계 문법** / 선이 거미줄처럼 되지 않게 **주연 연결 제한**.

## 25. 갈증의 심연 후속 문법 (방향만 · 미구현)

- 정체성: 마시는 자·낮은 액체·심연 핵·흡수구·웅덩이·생명/자원을 빨아들이는 흐름.
- 문법: target에서 보스 흡수구로 흐르는 **drain ribbon** / 바닥에서 퍼지는 낮은 웅덩이 데칼 / 직선 레이저보다 **말려 들어가는 곡선** / 흡수 성공 시 보스 코어·웅덩이 잠깐 팽창 / 막거나 정화되면 흐름 끊기고 잔광 사라짐 / **Actor를 완전히 덮는 액체 FX 금지**.

## 26. runtime 입력 후보 (읽기 전용 · 존재 미확정)

후속 구현은 **기존 전투 상태만 읽어 presentation으로 표현**. 입력 후보:
- 보스 cast 상태 / target id / shield 값·상태 / interrupt 성공 이벤트 / damage·block 결과 / 기존 render 사이클의 class·data-state.

## 27. 실제 코드 확인 필요 항목 (후속 구현 전 읽기 전용 검토)

- ★**아래는 "있다"고 단정하지 않는다** — 후속 카드 착수 전 실제 CORE/어댑터 코드를 읽어 확인해야 한다:
  - 파쇄자 smash의 target·shield 분기가 render에 노출되는 상태 플래그가 있는가
  - 도적 차단의 **실제 가능 시점·이벤트**가 runtime에 존재하는가(§22-5)
  - boss cast anchor에 해당하는 시전 상태/원점이 표현 가능한가
  - shield 값이 render 시점에 읽히는가
- 전사 sbFigPose(기존 상태만 읽는 포즈 연결)가 **선례**. 새 상태·타이머·판정을 만들지 않는다.

## 28. 금지선 (⛔)

- FX를 위한 새 전투 판정 생성 / 포즈를 위한 새 gameplay timer / 예고를 위한 패턴 시간 변경 / 행동선을 위한 target 선정 변경 / visual timeout이 실제 계산 지연 / 존재하지 않는 runtime 상태를 있다고 단정 / 행동선·데칼이 버튼을 가리키는 튜토리얼 선 / 귀여운 무대를 전술 도식판으로 바꿈 / 기존 renderFx·target ring·상태 플래시 제거·대체.
