## 4. 첫 실행 검증 루틴 (✅ — 51.4가 뜨기 전엔 아무것도 짓지 않는다)

이관 직후, 새 환경에서 순서대로:

① **바이트/라인/md5**: `wc -c index.html && wc -l index.html && md5sum index.html`
   → **106650 / 1756 / 34addd9c3441cf8c2e1f83e8303f0475** 정확 일치. 다르면 여기서 멈추고 어떤 버전인지 확인(재타이핑 ⛔).
② **코어 pristine 초기화**: `awk '/\/\/__CORE_START__/{f=1;next} /\/\/__CORE_END__/{f=0} f' index.html > core_pristine.js`
   → 이후 모든 카드의 byte-identical diff 기준(394줄/19,545 bytes 기대).
③ **node --check**(node 가능 시): script 태그 사이 추출 후 문법 검사(명령 세트는 세이브 v8 부록 B).
④ **무입력 스모크**: 세이브 v8 부록 A 하네스 복붙 재구축 → `defeat 51.4 (1,029 steps)` 확인. **이 숫자가 다르면 코어 오염 — 전면 중지.**
⑤ **390×844 overflow**: 브라우저/폰에서 전투 화면 세로 스크롤 0 · 스킬 6버튼 줄 넘침 0 (97 §11).
⑥ **보존 grep**: 기존 9종(차단! 1 · 차단 실패! 1 · renewUse 2 · seedOnHit 2 · chapelSaveReturn 2 · data-shell 2 · renderSortie 2 · pickReportHint 2 · nextThreatFor 2) + **focus 5종**(focusIsArmed 7 · focusUse 2 · focusWrapUse 2 · focusBtnUpdate 2 · FOCUS_REFUND 2).
